# SnapRep 完整测试文档

## 项目概述

**SnapRep** 是一个全栈健身应用，专注于"随地健身应用"的核心理念，让用户在任何地点使用身边物品完成针对性训练。

### 项目架构
- **后端**: NestJS + GraphQL + PostgreSQL + Supabase 混合架构
- **前端**: Flutter 跨平台移动应用
- **数据库**: PostgreSQL with Prisma ORM (Supabase托管)
- **认证**: Supabase Auth (匿名登录 + 邮箱登录)
- **存储**: Supabase Storage (结果卡片图片)
- **AI识别**: TensorFlow Lite (物体识别)

### 核心业务价值
- **目标用户**: 18-55岁普通人，想动但不想学专业动作
- **关键指标**: 从打开到得到3个动作建议≤30秒(P75)
- **核心功能**: 60秒快速开始、AI物品识别、成果卡片收集

### 核心性能要求
- **TTV (Time to Value)**: ≤30秒 从应用打开到获得锻炼推荐

- **API响应**: 大部分<500ms
- **会话完成率**: ≥95%

## 29个API端点测试

### Supabase Auto REST API (12个端点)

#### 1. 场景和器材 (4个)
- **GET /rest/v1/scenarios** - 获取场景列表
- **GET /rest/v1/equipment** - 获取器材列表
- **GET /rest/v1/scenarios/{id}** - 获取单个场景
- **GET /rest/v1/equipment/{id}** - 获取单个器材

#### 2. 训练会话 (3个)
- **POST /rest/v1/workout_sessions** - 创建训练会话
- **PATCH /rest/v1/workout_sessions/{id}** - 更新会话状态
- **GET /rest/v1/workout_sessions** - 获取训练历史

#### 3. 分享卡片 (3个)
- **POST /rest/v1/share_cards** - 保存卡片记录
- **GET /rest/v1/share_cards** - 获取我的卡片
- **GET /rest/v1/share_cards/{id}** - 获取卡片详情

#### 4. 用户和统计 (2个)
- **GET /rest/v1/users/{id}** - 获取用户信息
- **PATCH /rest/v1/users/{id}** - 更新用户设置

### NestJS Custom API (14个端点)

#### 1. 推荐系统 (3个)
- **POST /api/v1/recommendations/quick** - 快速推荐
- **POST /api/v1/recommendations/replace** - 替换单个动作
- **GET /api/v1/recommendations/alternatives** - 获取替换候选

#### 2. 卡片生成 (2个)
- **POST /api/v1/cards/generate** - 生成分享卡片
- **GET /api/v1/rarity/calculate/{code}** - 计算稀有度

#### 3. 主题周 (2个)
- **GET /api/v1/theme-weeks/current** - 获取当前主题周
- **POST /api/v1/theme-weeks/{id}/join** - 加入主题周

#### 4. AI识别 (1个)
- **POST /api/v1/ai/recognize-equipment** - AI物体识别

#### 5. 用户分析 (6个)
- **PATCH /api/v1/analytics/users/{id}** - 更新用户分析数据
- **GET /api/v1/analytics/users/{id}/funnel** - 获取漏斗状态
- **GET /api/v1/analytics/users/{id}/metrics** - 获取用户指标概览
- **GET /api/v1/analytics/users/{id}/daily** - 获取每日指标
- **GET /api/v1/analytics/cohorts** - 获取群组分析

## 性能验证测试

### 关键性能指标
| 指标 | 目标 | 测试方法 | 验证点 |
|------|------|----------|--------|
| **TTV** | ≤30秒 | E2E完整流程计时 | 从打开到得到推荐 |
