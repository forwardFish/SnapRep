# 前后端枚举同步指南

## 核心原则

**后端 Prisma Schema 是唯一的数据源，前端手动同步。**

---

## 一、后端枚举定义位置（唯一数据源）

### 文件位置
```
backend/prisma/schema.prisma
```

### 所有枚举定义

## 二、前端需要同步的文件

### 1. 肌肉群枚举 (PrimaryMuscle → TargetMuscle)

**前端文件**: `frontend/lib/core/models/target_muscle.dart`

**同步规则**:
- 前端枚举值的 `@JsonValue` 必须与后端 `PrimaryMuscle` 完全一致
- `code` 字段用于 API 调用，必须匹配后端
- `displayName` 仅用于 UI 显示，可以自定义

**示例**:
```dart
enum TargetMuscle {
  @JsonValue('CHEST')
  chest('CHEST', 'Chest', 'Build chest muscles', '💪'),

  @JsonValue('BACK')
  back('BACK', 'Back', 'Strengthen back', '🏋️'),

  @JsonValue('CHEST_BACK')
  chestBack('CHEST_BACK', 'Chest & Back', 'Improve posture', '🤸'),

  @JsonValue('LEGS')
  legs('LEGS', 'Legs', 'Build lower body power', '🦵'),

  @JsonValue('GLUTES')
  glutes('GLUTES', 'Glutes', 'Shape and tighten', '🍑'),

  @JsonValue('CALVES')
  calves('CALVES', 'Calves', 'Define leg lines', '🦶'),

  @JsonValue('SHOULDERS')
  shoulders('SHOULDERS', 'Shoulders', 'Shoulder strength', '💪'),

  @JsonValue('ARMS')
  arms('ARMS', 'Arms', 'Sculpt arm definition', '💪'),

  @JsonValue('CORE')
  core('CORE', 'Core', 'Strengthen body center', '⚡'),

  @JsonValue('FULL_BODY')
  fullBody('FULL_BODY', 'Full Body', 'Comprehensive training', '🏃'),

  @JsonValue('NECK_SHOULDER')
  neckShoulder('NECK_SHOULDER', 'Neck & Shoulders', 'Relieve office stress', '💆');

  const TargetMuscle(this.code, this.displayName, this.description, this.emoji);

  final String code;         // ← 必须与后端枚举值一致
  final String displayName;  // ← 可以自定义
  final String description;
  final String emoji;
}
```

**⚠️ 重要**: 调用 API 时必须使用 `.code`:
```dart
// ✅ 正确
'targetMuscles': muscles.map((m) => m.code).toList()

// ❌ 错误
'targetMuscles': muscles.map((m) => m.displayName).toList()
```

## 三、手动同步流程

### 当后端添加/修改枚举时

#### 步骤 1: 修改后端
编辑 `backend/prisma/schema.prisma`，添加或修改枚举值

```prisma
enum PrimaryMuscle {
  // ... 现有值
  NEW_MUSCLE  // 新增的肌群
}
```

#### 步骤 2: 运行 Prisma 命令
```bash
cd backend
npx prisma generate              # 生成 Prisma Client
npx prisma migrate dev --name add_new_muscle  # 创建数据库迁移（可选）
```

#### 步骤 3: 手动同步到前端
打开对应的前端文件，添加新枚举值：

```dart
// frontend/lib/core/models/target_muscle.dart
enum TargetMuscle {
  // ... 现有值

  @JsonValue('NEW_MUSCLE')
  newMuscle('NEW_MUSCLE', 'New Muscle', 'Description', '💪'),
}
```

#### 步骤 4: 重新生成 Dart 代码
```bash
cd frontend
flutter pub run build_runner build --delete-conflicting-outputs
```

## 五、验证同步是否正确

### 检查清单

- [ ] 后端 `schema.prisma` 中的所有枚举值都在前端有对应
- [ ] 前端 Dart enum 使用了 `@JsonValue('BACKEND_VALUE')`
- [ ] 前端 API 调用使用 `.code` 而不是 `.displayName`
- [ ] 运行 `npx prisma generate` 成功
- [ ] 运行 `flutter pub run build_runner build` 成功
- [ ] 前后端都能正常编译

### 测试方法

1. **后端测试**: 启动后端，确保编译无错误
   ```bash
   cd backend
   npm run start:dev
   ```

2. **前端测试**: 运行前端应用，测试 API 调用
   ```bash
   cd frontend
   flutter run
   ```

3. **API 测试**: 检查前端发送的请求参数是否正确
   - 打开 Flutter DevTools
   - 查看网络请求
   - 确认 `targetMuscles` 等字段使用的是枚举 code 值

## 六、常见错误和解决方案

### 错误 1: `each value in targetMuscles must be a valid enum value`

**原因**: 前端发送的枚举值后端不认识

**解决**:
1. 检查后端 `schema.prisma` 是否包含该值
2. 确认前端使用 `muscle.code` 而不是 `muscle.displayName`
3. 确认前端 `@JsonValue` 与后端完全一致

### 错误 2: 前端编译错误 `Undefined name`

**原因**: 前端 Dart enum 缺少后端的某个值

**解决**:
1. 对照 `schema.prisma` 添加缺失的枚举值
2. 运行 `flutter pub run build_runner build`

### 错误 3: Prisma generate 失败

**原因**: `schema.prisma` 语法错误

**解决**:
1. 检查枚举定义语法
2. 确保没有重复的枚举值
3. 确保注释使用 `//` 而不是 `/* */`

## 七、部署注意事项

### 后端部署
- 确保 `backend/prisma/schema.prisma` 文件在部署包中
- 运行 `npx prisma generate` 生成客户端
- 如有数据库变更，运行 `npx prisma migrate deploy`

### 前端部署
- Flutter 应用编译时会包含所有 Dart 代码
- 无需额外配置
- 确保在构建前运行过 `flutter pub run build_runner build`

---

## 八、总结

### 唯一数据源
**`backend/prisma/schema.prisma`** 是所有枚举的唯一定义源

### 同步方式
手动复制枚举值到前端对应文件，使用 `@JsonValue` 确保序列化正确

### 核心规则
1. 后端改了枚举 → 前端必须手动同步
2. 前端调用 API 必须使用 `.code` 字段
3. `@JsonValue` 必须与后端枚举值完全一致
