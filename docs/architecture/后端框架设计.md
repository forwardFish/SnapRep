0) 本文用途

这是一份给 Claude Code 直接开工的后端开发说明：包含目录结构、环境变量、鉴权、Prisma schema、API 合约（DTO/错误格式）、推荐算法、种子数据、部署与验收标准。不需要额外确认即可按本文实现 P0.5。

1) 业务概述（与客户端契合）

目标用户：想动但不想学、不去健身房、无器械的大多数人群。

使用路径：3 步引导（意图→场景/物品→部位）→ 结果页确认 3×20s → 跟练 60 秒 → 完成反馈。

后端职责（MVP）：

生成 60 秒方案（3 张动作卡，含预览、要点、标签）

管理动作库（白名单）与快照

Deeplink 短链

支付回调（订阅标记）

事件采集/统计（轻量）

2) 技术栈与基座改造

基座仓库：notiz-dev/nestjs-prisma-starter

保留：REST + Swagger + Prisma + Docker

关闭：GraphQL（如模板默认启用，先注释掉）

数据库：Prisma 指向 Supabase（见 §4 连接）

鉴权：Supabase JWT 直透（后端只校验，不自建登录）

媒体：Supabase Storage 私有桶 + 签名 URL（客户端访问）

4) 环境变量与连接（Supabase 双连接串）

.env（开发/部署都使用；分别注入正确值）

# App
PORT=8080
NODE_ENV=development
APP_NAME=aifit-api
APP_ORIGIN=https://your-webapp.example

# Swagger
SWAGGER_ENABLED=true

# Supabase JWT 校验
SUPABASE_JWT_SECRET=your-supabase-jwt-secret  # Project Settings -> JWT

# Prisma 连接（强烈建议双连接）
DATABASE_URL="postgresql://user:pass@db-pooler.supabase.co:6543/postgres?pgbouncer=true&connection_limit=1&sslmode=require"
DIRECT_URL="postgresql://user:pass@db.supabase.co:5432/postgres?sslmode=require"

# 短链域名（可与主域一致）
DEEPLINK_DOMAIN=https://aifit.link

# Stripe（回调用）
STRIPE_WEBHOOK_SECRET=whsec_***


prisma/schema.prisma 里 datasource：

datasource db {
  provider  = "postgresql"
  url       = env("DATABASE_URL")   // 运行时走 pooler
  directUrl = env("DIRECT_URL")     // 迁移时走直连
}
generator client {
  provider = "prisma-client-js"
}


注意：迁移用 npx prisma migrate deploy（或 dev 环境 migrate dev），不要通过 pooler 执行迁移，避免事务模式问题。

5) 鉴权与安全

客户端（Flutter/Web）从 Supabase 拿 access_token → Authorization: Bearer <token>。

后端 SupabaseJwtAuthGuard 使用 SUPABASE_JWT_SECRET 校验；解出 sub（auth.uid()）作为 userId。

RLS：客户端直连 Supabase 的表仍可保留 RLS；后端模块（Webhook/管理）使用 service_role 或服务器端连接。

统一错误响应：

{ "code": "BadRequest", "message": "details...", "details": { "field": "reason" } }


基础安全：Helmet、CORS 白名单、ValidationPipe（whitelist:true, forbidNonWhitelisted:true）、速率限制（可在 API Gateway 层或 Nest 中间件实现）。
