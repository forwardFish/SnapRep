## 业务流程总览

### 产品核心理念

**SnapRep = "随地健身应用"**
- **核心价值**: 任何地点、用手边物件完成针对性训练,60秒快速开始
- **目标用户**: 18-55岁普通人,想动但不想学专业动作
- **关键指标**: 从打开到得到3个动作建议≤30秒(P75)

### 架构说明

- **后端**: NestJS + Supabase 混合架构
- **数据库**: PostgreSQL (Supabase托管)
- **认证**: Supabase Auth (匿名登录 + 邮箱登录)
- **存储**: Supabase Storage (结果卡片图片)
- **AI识别**: TensorFlow Lite (物体识别)

### 5大核心页面

```
┌─────────────────────────────────────────┐
│ 1. 首页 (HomePage)                      │
│    - Hero CTA "给我60秒"                 │
```

## 详细业务流程

### 流程1: 用户认证与首次进入

#### 1.1 应用启动与匿名登录

**用户操作**:
1. 用户首次打开SnapRep应用
2. 看到启动画面(Splash Screen)
3. 系统自动执行匿名登录

**系统处理**:
```
前端 → 调用 Supabase Auth API
     → supabase.auth.signInAnonymously()
     → 返回匿名用户凭证
     → 存储JWT Token到本地
     → 创建基础用户记录
     → 进入首页
```

**涉及API**:
- Supabase Auth: `signInAnonymously()`
- 自动创建: `POST /rest/v1/users` (通过RLS自动触发)

**预期结果**:
- 用户获得匿名身份
- JWT Token存储在本地
- 可以使用所有核心功能
- 后续可升级为邮箱账号

**数据流**:
```json
// Supabase Auth Response
{
  "access_token": "eyJ...",
  "refresh_token": "...",
  "user": {
    "id": "uuid-anon-user-123",
    "is_anonymous": true,
    "created_at": "2024-10-30T10:00:00Z"
  }
}

// Users表自动创建记录
{
  "id": "uuid-anon-user-123",
  "email": null,
  "name": null,
  "is_anonymous": true,
  "total_workouts": 0,
  "current_streak": 0
}
```

---

#### 1.2 邮箱注册/登录(可选)

**用户操作**:
1. 用户在"我的"页面点击"绑定邮箱"
2. 输入邮箱地址
3. 接收Magic Link邮件
4. 点击邮件中的链接
5. 自动完成登录

**系统处理**:
```
前端 → supabase.auth.signInWithOtp(email)
     → Supabase发送Magic Link邮件
     → 用户点击链接
     → 回调到应用
     → 匿名账号升级为正式账号
     → 历史数据保留
```

**涉及API**:
- Supabase Auth: `signInWithOtp(email)`
- 升级账号: `PATCH /rest/v1/users/{id}`

**预期结果**:
- 匿名账号升级为正式账号
- 所有历史数据保留
- 支持多设备同步
- 可以分享和复刻

**缓存策略**:

- 器材列表: TTL 1天

#### 2.2 快速开始 - "给我60秒"

**用户操作**:
1. 点击首页的"给我60秒"按钮
2. 系统立即生成推荐
3. 跳转到动作结果页

**系统处理**:
```
前端 → POST /api/v1/recommendations/quick
     → Body: {
         "intent": "STRETCH",  // 默认:舒展筋骨
         "equipment": ["hands_free"],  // 默认:空手
         "duration": 60  // 默认:60秒

     → 返回3个动作 + 创建workout_session
     → 跳转到动作结果页
```

**涉及API**:
- `POST /api/v1/recommendations/quick`

**推荐算法逻辑**:
```typescript
// NestJS Service核心算法

2. 筛选符合条件的动作 (意图+器材+场景)

   - 避免最近7天重复动作
   - 难度递进 (简单→中等→稍难)
   - 部位均衡覆盖
   - 时长匹配 (总和60秒±10%)
   - 安全优先 (过滤禁忌症)
4. 选择Top 3动作
5. 创建训练会话记录
6. 生成替换候选列表 (6-9个)
```

#### 2.5 AI物品识别

**用户操作**:
1. 点击首页的"拍照"图标
2. 授权相机权限
3. 拍摄周围物品
4. 等待识别结果(≤3秒)
5. 查看识别出的物品
6. 确认或修改后继续

**系统处理**:
```
前端 → 调用相机拍照
     → 上传图片到 POST /api/v1/ai/recognize-equipment
     → NestJS调用TensorFlow Lite模型
     → 物体检测与识别
     → 返回识别结果(按置信度排序)
     → 如果置信度≥85%: 自动预选并进入引导页
     → 如果置信度<85%: 显示候选列表,用户手选
```

**涉及API**:
- `POST /api/v1/ai/recognize-equipment`

**请求格式**:
```typescript
// multipart/form-data
{
  image: File,
  confidence: 0.85,  // 可选
  maxResults: 5      // 可选
}
```

**响应示例**:
```json
{
  "success": true,
  "recognized": [
    {
      "equipmentCode": "chair",
      "confidence": 0.92,
      "boundingBox": {
        "x": 100, "y": 150,
        "width": 200, "height": 300
      },
      "equipmentInfo": {
        "id": "eq-001",
        "name": "Chair",
        "iconUrl": "https://..."
      }
    },
    {
      "equipmentCode": "wall",
      "confidence": 0.88,
      ...
    }
  ],
  "fallbackSuggestions": [
    {
      "equipmentCode": "sofa",
      "name": "Sofa",
      "reason": "Similar to detected object"
    }
  ]
}
```

**性能目标**: AI识别≤3秒

**容错机制**:
- 权限被拒绝 → 引导手动选择物品
- 识别失败 → 回退到九宫格
- 离线模式 → 显示"需要网络连接"提示

### 流程3: 锻炼引导3步骤

#### 3.1 Step 1: 运动意图选择

**页面目标**: 快速捕捉用户今天想怎么动

**用户操作**:
1. 看到4个选项卡片:
   - 放松 (降紧张/舒缓神经) 🧘
   - 舒展筋骨 (拉伸与活动度) 🤸
   - 适当运动 (轻汗/微心率) 🏃
   - 主体锻炼 (轻力量/稳定性) 💪
2. 可多选≤2项
3. 点击"下一步"或"直接开练60秒"

**默认值**: "舒展筋骨" (最通用)

**快捷路径**:
- 点击"直接开练60秒" → 跳过后续步骤,直接生成推荐

**前端实现**:
```dart
// 意图选项 (前端hardcoded)
enum WorkoutIntent {
  RELAX,         // 放松
  STRETCH,       // 舒展筋骨
  LIGHT_CARDIO,  // 适当运动
  STRENGTH       // 主体锻炼
}

// 用户选择
List<WorkoutIntent> selectedIntents = [WorkoutIntent.STRETCH];
```

**交互细节**:
- 支持多选≤2项
- 选中有轻弹反馈(150ms)
- "跳过"使用默认值继续

---

#### 3.2 Step 2: 场景+物品选择

**页面目标**: 确定用户所在环境和可用物品

**用户操作**:
1. 选择场景 (单选):
   - 家 / 办公室 / 旅途 / 公园 / 其他
2. 选择物品 (多选):
   - 场景切换时,物品建议会动态更新
   - 可选"无"(空手) 或 具体物品
3. 可选: 点击"拍摄识别"
4. 点击"下一步"

**场景与物品联动**:
```typescript
// 场景切换时更新物品建议
onScenarioChanged('office') {
  // 办公室场景推荐: 椅子, 墙面, 水瓶, 书包
  suggestedEquipment = ['chair', 'wall', 'bottle', 'backpack'];
}

onScenarioChanged('park') {
  // 公园场景推荐: 长椅, 台阶, 树, 栏杆
  suggestedEquipment = ['bench', 'stairs', 'tree', 'railing'];
}
```

**涉及API**:
- 场景列表: `GET /rest/v1/scenarios`
- 器材列表: `GET /rest/v1/equipment`
- (可选) AI识别: `POST /api/v1/ai/recognize-equipment`

**默认值**:
- 场景: "家" (最通用)
- 物品: "无" (空手)

---

#### 3.3 Step 3: 目标部位选择

**页面目标**: 明确用户想重点锻炼的部位

**用户操作**:
1. 看到部位选项 (2列Chip栅格):
   - 全身 / 颈肩 / 胸背 / 核心
   - 大腿 / 臀部 / 小腿 / 手臂
2. 可多选≤2项
3. 点击"完成引导"

**默认值**: "全身" (最通用)

**前端实现**:
```dart
// 目标部位 (前端hardcoded)
enum TargetMuscle {

```

**完成引导**:
```
前端 → 汇总所有选择:
     {
       intent: ["STRETCH"],
       equipment: ["chair", "wall"],

       targetMuscles: ["NECK_SHOULDER"]
     }
     → POST /api/v1/recommendations/quick
     → 生成3个动作
     → 跳转到动作结果页
```

#### 5.3 稀有度系统说明

**5档稀有度**:
1. **COMMON (常见)** - 灰白简洁, 频次>40%
2. **UNCOMMON (进阶)** - 柔色渐变, 频次15-40%
3. **RARE (稀有)** - 斜向金属光泽, 频次5-15%
4. **EPIC (史诗)** - 炫彩棱镜/全息纹, 频次1-5%
5. **LEGENDARY (传奇)** - 动态细闪+细密纹理, 频次<1%

**计算规则**:
```typescript
// 基于7日全球频次
const score = 本器材7日使用次数 / 所有器材7日总使用次数;

if (score <= 0.01) return 'LEGENDARY';
else if (score <= 0.05) return 'EPIC';
else if (score <= 0.15) return 'RARE';
else if (score <= 0.40) return 'UNCOMMON';
else return 'COMMON';
```

**器材系列**:
- 家具系 (椅子/沙发/桌子)
- 墙面系 (墙面/门/栏杆)
- 瓶罐系 (水瓶/矿泉水瓶)
- 背包携行 (背包/手提袋)
- 台阶座椅 (台阶/长椅)
- ... 共10大系列

**特殊标签**:
- 静音完成 (所有动作都是静音)
- N-Day Streak (连续N天)
- 城市限定 (如"Beijing Limited")
- 主题周 (如"#椅子日")

## 总结

### 核心业务流程回顾

SnapRep应用的核心业务流程可以总结为以下7个主要环节:

1. **认证进入** - 匿名登录/邮箱登录
2. **快速启动** - 首页"给我60秒"或场景/物品选择
3. **引导流程** - 3步引导(意图→场景物品→部位)
4. **动作推荐** - 智能算法生成3个动作
5. **跟练执行** - 逐个完成动作
6. **成果生成** - 生成稀有度卡片
7. **分享收集** - 分享到社交平台,收集图鉴

### 关键性能指标

- **TTV (Time to Value)**: ≤30秒
- **AI识别**: ≤3秒
- **推荐生成**: ≤5秒
- **卡片生成**: ≤800ms
- **API响应**: 大部分<500ms

### 数据流向

```
用户输入 → NestJS/Supabase → PostgreSQL → 推荐算法
                                          → 稀有度计算
                                          → 卡片生成
        → Supabase Storage (图片)
        → 返回前端
```
