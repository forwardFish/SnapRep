0. 项目综述
电梯陈述
给我 60 秒，别学动作也能舒服一点。
这是一款为想动但不想学、没有器械、不去健身房的普通人打造的 “随地可练 · 动作微练” App：打开就出几个安全动作，做完一键生成可晒成果卡，支持接龙复刻与轻社交。

0.1 产品定位（一句话）

一款让用户在“任何地点、用手边现有物件”完成针对性训练的轻量健身 App。
让普通用户在任何环境、零学习成本下，在“任何地点、用手边现有物件”完成针对性训练的轻量健身，并生成可分享的成果卡，形成“打开 → 行动 → 分享”的高转化闭环。

目标人群

画像：

18–55 岁普通人；久坐上班族/学生/宝妈/轻旅行者

诉求：想活动筋骨、缓解不适/提神放松，但不愿学专业动作、不购置器械、不去健身房

地域：全球化产品，英语 + 中文优先；首发以 北美/欧洲/港台 为主（付费意愿与在家/随地运动渗透更高）

设备：Android / 鸿蒙优先上线；离线可用、权限可选，适配中低端机型（低耗时、轻交互）

0.2 要解决的问题

许多人没有器械/没有时间，但希望随时动一下；传统课程型 App 起步成本高、需要预设器材与完整时段。

用户不知道“在当下环境，用手边的东西，能练哪几个部位、做哪些安全动作”。

0.3 解决思路（价值主张）

地点无关：家、办公室、公园、旅途都能练。

物件友好：把“椅子、背包、水瓶、台阶、毛巾、书等”映射为安全动作清单。

目标清晰：从目标/部位出发（如“练臀腿/练背/燃脂 5 分钟”），结合可用物件给出几个 可执行动作（三要点+两条安全红线+剂量）。

可分享的完成感：一键生成成果卡，促进复练与传播。

0.4 差异化原则

Anywhere First：先保证“无器械/弱网/碎片时间”也能完成一次有效训练。

Object-Aware：物件识别与选择是辅助手段，帮助用户更快把“环境→动作”。

秒级上手：从打开到得到 3 个动作建议≤30 秒（P75）。

1. 目标用户与场景
1.1 目标用户

随地练大众：无器械、时间碎片化、基础一般（新手或回归者）。

次级：办公室久坐人群、差旅/学生、轻体态改善需求者。

1.2 核心场景（按进入方式）

部位优先：我想练某个部位/目标（如“核心 5 分钟/背部激活”）→ 选择我手边有的物件→ 得到 3 个动作。

物件优先：看到手边的物件（拍照或手选）→ 选择要练的部位/目标→ 得到 3 个动作。

空手备选：暂时没有合适物件 → 选择空手替代动作（仍给 3 个）。

1.3 用户痛点

启动门槛高：要选课、看教学、搬器材 → 最终不开始。

时间碎片化：只有 1–3 分钟；长课程直接放弃。

怕累/怕出汗/怕酸痛：一次糟糕体验会“劝退”。

怕尴尬：不愿在公开场合做夸张动作；也怕被“身材评判”。

信息过载：术语、动作细节太多，认知负担重。

难以坚持：无反馈、无成就、无社交轻激励。

隐私与安全：不想上传人像，不想有受伤风险。

1.4 核心诉求（对应产品要点）

“马上舒服一点”：当场能缓解/提神，而不是“学会一个动作”。

产品：3×20s 绿档起步；两条红线+“更稳替代”；办公室/沙发隐身动作优先。

“零学习成本”：不看教程、不选课程，也能立刻开始。

产品：首页一个最大按钮「给我 60 秒」+ 九宫格物件，拍照可选但非必需。

“随地可练”：家里、办公室、旅行途中都能做。

产品：物件库先打磨 6 类（椅子/墙面/水瓶/背包/台阶/沙发），每类有绿/蓝/红爬梯。

“好玩、可晒、无评判”：不暴露身材、不卷成绩，但能显得聪明有趣。

产品：成果卡只含“物件×3 动作 + 日期 + #随地可练”，不含体重/身材；趣味皮肤；

一键复刻深链与“第 N 棒接龙”，带来轻社交而非压力。

“轻理由再回来”：每天给一个“刚好能做”的借口。

产品：完成后首页今日卡（一键再练）+ 惊喜我（没试过的物件）+ ≤1 条/天场景提醒。

“不花心思也能进步”：有看得见的成就，但不强制计划表。

产品：绿→蓝→红解锁边框、家伙库点亮、Bingo 九宫格；周目标小条代替课程日历。

2. 用户旅程（高层流程）

选择入口

A：练哪个部位/目标（胸/背/腿/臀/肩/臂/核心/燃脂·时长）

B：我手边有什么（拍照辅助 or 手选九宫格）

确认“可用物件”（可多选：椅子/背包/水瓶/台阶/毛巾/书/门框/扫把/长凳/米袋…/空手）

得到 3 个动作卡（每卡：标题、主肌群/难度、三要点、两红线、剂量；可“换一组”）

生成成果卡（9:16，可保存/分享）

备注：拍照仅出现在步骤 2 作为加速器；用户也可完全不拍照而完成流程。

3. 范围与优先级（MVP 视角）
3.1 Must-Have（P0）

入口 A：部位/目标选择（必有）

入口 B：可用物件选择

方式一：手选九宫格（首发 10 类常见物件+空手）

方式二：拍照辅助识别（可选用/可跳过）

动作推荐（3 卡）：来自安全白名单，含三要点+两红线+剂量，支持“换一组”。

成果卡：9:16 专业简洁版；一键保存/系统分享。

离线可用、权限拒绝兜底、基础埋点与性能指标。

3.2 Should-Have（P1）

“今日 2–5 分钟”快捷卡（按部位/目标与可用物件即时生成）。

最近常用物件/部位快捷记忆。

英文语言包；基础本地化（单位/文案）。

3.3 Won’t-Have（此阶段排除）

登录/订阅/社区/完整训练计时器/姿态纠错/自动周计划/自研后端。

4. 功能需求（无代码，规则导向）
4.1 入口 A：部位/目标选择

部位：胸/背/腿/臀/肩/臂/核心/全身

目标：激活/燃脂（时长 2/5/10 分钟）/体态微调（P1）

选择后进入可用物件选择页。

4.2 入口 B：可用物件选择

手选九宫格（首发 10 类 + 空手）：椅子/背包/水瓶/毛巾/书/台阶/门框/扫把/长凳/米袋/空手

拍照辅助：拍一张；若识别成功则预选对应物件；识别不确定直接回落到手选页。

可多选物件（如：椅子 + 水瓶）。

4.3 动作推荐（3 卡）

输入：部位/目标 + 可用物件（≥1）

规则：

在“物件×部位”的白名单动作库中检索；

优先低风险（初级/稳定支撑），避免连续同关节高负荷；

同一轮 3 个动作不重复主肌群；

若“空手”在可用物件内，可混出 1–2 个空手替代动作。

每卡字段：标题、主肌群/难度、三要点、两红线、剂量（次数×组 or 秒×组）、备注（可选重量提示）。

交互：支持“换一组”（从未出过的候选里重新抽取，不重复）。

4.4 安全与提示

红线库：为每个动作维护 2 条硬禁忌（如“膝盖不内扣/脊柱中立/避免耸肩”）。

重量/强度提示：如“水瓶 1L≈1kg；背包≤体重 5–10% 起步”。

免责声明：一般健康建议，非医疗；不适请停止。

4.5 成果卡（生成与分享）

内容：物件图（若有拍照则用）、3 动作（标题+剂量）、日期、话题（#随地可练）、品牌字样。

尺寸：9:16；支持保存到相册与系统分享面板。

风格：专业简洁（P1 再加趣味版）。

4.6 兜底与无网

识别置信度不足→自动弹出手选九宫格。

无网仍可全流程完成；分享失败时提示“已保存到相册”。

5. 内容与规则规范（供内容/教练同学执行）
5.1 物件分类（首发建议覆盖）

椅子 / 背包 / 水瓶 / 毛巾 / 书 / 台阶/楼梯 / 门框/墙 / 扫把/杆 / 长凳/护栏 / 米袋/沙袋 / 空手

5.2 部位标签

胸 / 背 / 腿 / 臀 / 肩 / 臂 / 核心 / 全身

5.3 动作条目规范（每条必填）

标题（中文 12 字内；英文 35 字符内）

主肌群（1–2）与难度（初/中）

三要点（每行≤12 字；动作/轨迹/呼吸）

两红线（硬禁忌）

剂量（次数×组 或 秒×组；默认 12×3 / 30s×3 可根据动作设定）

备注（可选：重量提示/空间限制）

5.4 组合规则小抄（给策划/算法使用）

组合 3 个动作时尽量覆盖不同肌群；

同一轮避免同关节高剪切力动作叠加（例如连续深蹲 + 跳箱）；

在“燃脂”目标下，允许加入 1 个心肺提升动作（如登阶/高抬腿），其余 2 个动作为抗阻或全身复合。

6. 成功指标（KPI）

北极星：一次会话成功获得 3 动作并生成成果卡的完成率
体验：TTV ≤ 30s（P75）；识别-手选综合命中 ≥85%
增长：分享率 ≥15%（内测）
质量：用户对“易懂/安全”的主观评分 ≥4.5/5（小样本问卷）

7. 非功能需求（NFR）

离线优先：无网络仍可完成覆盖流程。

性能：从选择入口到拿到 3 动作 ≤ 5s；成果卡导出 ≤ 800ms。

可访问性：主要按钮≥44dp；字号≥14sp；色彩对比≥4.5:1。

隐私：图片仅用于本地识别与卡片生成，默认不上传。

国际化准备：文案与单位可配置，保留英文占位。

8. 风险与应对
| 风险          | 影响   | 对策                                 |
| ----------- | ---- | ---------------------------------- |
| 用户误以为“必须拍照” | 放弃使用 | 在入口明确“可直接选择物件/空手”，拍照标注为“快速识别（可跳过）” |
| 识别不稳定       | 流程中断 | 置信阈值+九宫格兜底；让“手选”成为第一等公民            |
| 动作安全顾虑      | 口碑风险 | 白名单+两红线；默认给“低风险/稳定动作”优先级           |
| 选择困难/信息过载   | 体验变慢 | 固定“只给 3 个动作”，支持“换一组”，不出现长列表        |
| 无网/权限拒绝     | 流程失败 | 离线可用；拒权回落到“手选”，并解释用途               |

9. MVP 验收标准（P0）

定位符合：无需拍照也能完成“部位/物件 → 3 动作 → 成果卡”闭环；拍照仅作为辅助。

时间：从打开到生成成果卡 ≤ 30s（P75）。

可得性：识别+手选综合成功率 ≥ 85%。

安全清晰：每卡均含“三要点+两红线+剂量”，无缺项。

可分享：9:16 成果卡生成成功率 ≥ 95%。

稳定：连续 20 次闭环无崩溃；无网也可完成流程。
