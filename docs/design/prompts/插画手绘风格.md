## 2. 新视觉风格 (大胆手绘风格 Bold Hand-Drawn Style)

*   **核心元素风格:**
    *   **表现力手绘插画:** 界面设计的灵魂。采用**风格鲜明、线条粗犷或富有表现力（如墨水、炭笔、马克笔或水彩质感）的手绘插画**。允许不完美的线条、随意的涂色或肌理。插画内容可以更抽象、更有力量感，围绕能量、梦想、突破等主题，营造独特、充满个性的视觉体验。重点应用于**启动页、大型卡片背景、空状态、成就徽章**等。
    *   **有机形态与纹理:** UI元素（卡片、按钮）可以有**轻微的不规则边缘或手绘轮廓线**，圆角可以不完全平滑。广泛运用**纸张纹理、画布纹理、墨迹、水彩晕染**等作为背景或元素填充，增强手作感。
*   **背景与层级:**
    *   **有质感的背景:** 整体页面背景采用**浅色但带有明显纹理**的底色（如米白色画布纹理 `#F8F4EC`、做旧纸张效果）或**柔和的、不均匀的暖色水彩/油画背景**。
    *   **对比与阴影:** 通过**粗线条轮廓、色彩对比**或**模拟手绘阴影（如交叉排线、深色墨迹区域）**来区分层级。避免使用标准的、平滑的投影，追求更自然的堆叠感。
*   **配色方案:**
    *   **主导色:** 可以是**相对饱和、对比强烈的色彩组合**，灵感可以来自水彩、油画颜料或彩色墨水。例如，使用**深蓝 (`#2A3A68`)、活力橙 (`#FF8C42`)、草木绿 (`#5F7A3C`)、浆果红 (`#A63D40`)** 等组合。颜色应用可以**不均匀，带有笔触感或晕染效果**。
    *   **背景色:** 带有纹理的浅色（见上）。
    *   **主文字:** **深褐色 (`#4E3B31`)、深灰蓝 (`#3B4A54`) 或近黑色 (`#222222`)**，确保在纹理背景上的**清晰度**。可考虑使用略带手写感的字体。
    *   **次要文字/提示文字:** 中性灰色 (`#777777`) 或与主文字同色系的浅色。
    *   **强调色/交互色:** 选择一种**非常鲜明、醒目的颜色**（如亮黄 `#FFD700`、亮粉 `#F72585` 或鲜艳的青色 `#4CC9F0`）用于交互状态、图表线条、重要提醒，形成视觉焦点。
*   **手绘感字体排版:**
    *   字体:
        *   **标题/关键标签:** 优先使用**表现力强的手写体或手绘风格字体** (需要确保中文支持和可读性，如某些特定的书法字体、手札体，或模拟效果的字体)，例如 `站酷手书体`、`庞门正道标题体` (如果风格匹配)。
        *   **正文:** 保证可读性，可选**略带书写感的无衬线体**或**结构清晰的宋体/黑体**（如 `思源宋体`, `思源黑体`）。
    *   显示：文字显示为中文。
    *   层级: (字号可根据字体特性微调)
        *   大标题: `32px` Bold/Heavy, (根据字体选择颜色，确保对比)。
        *   卡片标题/重要标签: `22px` Medium/Bold, (深色文字)。
        *   正文/列表项: `16px` Regular, (深色文字)。
        *   小字号/说明文字: `14px` Regular, (中性灰色)。
*   **表单优化:**
    *   输入框 (`<input>`, `<textarea>`): **清晰的手绘边框**（如模拟墨水勾勒，`border: 2px solid #4E3B31;`），或**无边框但有明显的纹理背景填充**（如浅色纸张）。圆角可以**略显随意，不完全对称**。
    *   焦点状态 (`:focus`): 边框**加粗或变为鲜明的强调色墨水线条**，或添加**类似手绘高光的简单标记**。
*   **交互动效:**
    *   **按钮:** 点击时可以有**轻微的抖动、颜色加深或模拟按压的形变**。悬停时可能有**边框线条加粗或颜色变化**。
    *   **卡片:** 悬停时触发**轻微的倾斜或位移**，以及**模拟手绘阴影的加深**。
    *   **动效风格:** 动画可以**不追求极致平滑**，允许带有一些**跳跃感或帧动画**的感觉，模拟翻页、笔触绘制等效果。

## 3. 技术规格

*   **输出格式:** 单个完整的HTML文件。所有CSS样式使用 `<style>` 标签内嵌，必要的简单JS（如图表动画触发）也内嵌在 `<script>` 标签中。
*   **布局:**
    *   HTML Body内创建一个容器，用于横向排列所有页面预览。
    *   每行展示 **6个** 页面预览。使用 Flexbox 或 Grid 布局实现。
    *   当页面数量超过6个时，自动换行。
*   **页面预览画板:**
    *   每个页面预览模拟手机屏幕，尺寸严格为 **375x812px**。
    *   **内部背景:** 每个画板的**模拟屏幕背景色**应设置为**带有纹理的浅色背景** (`background-color: #F8F4EC;` 或类似，最好能用CSS模拟简单纹理）。
    *   **画板边框:** 为每个画板添加 **1px 的深灰色描边** (`border: 1px solid #555555;`) 以便在浏览器中清晰看到画板边界。
    *   画板之间需要有适当的间距 (`margin`)。
*   **矢量图标系统:**
    *   **必须** 使用SVG `<symbol>` 技术定义图标。在HTML文件顶部 `<svg>` 块中定义所有需要的图标。图标设计需为**清晰的手绘风格（如粗线条、简笔画、墨水笔触感）**，避免过于精细和复杂。
    *   在页面中通过 `<svg><use xlink:href="#icon-name"></use></svg>` 的方式引用图标。图标颜色应能通过CSS `fill` 或 `stroke` 控制，通常使用**深色文字颜色** (`#4E3B31`) 或**强调色**。
*   **动态折线图:**
    *   在“目标详情页”的进度追踪模块中实现。
    *   使用SVG `<path>` 绘制折线。线条**模拟手绘笔触（可通过CSS filter或SVG filter实现轻微抖动/粗细变化，或使用虚线模拟）**，颜色使用**鲜明的强调色**。
    *   **必须** 使用CSS `stroke-dasharray` 和 `stroke-dashoffset` 属性，并配合 `@keyframes` 动画，实现线条动态绘制效果。
*   **卡片效果:**
    *   应用于主页、目标列表等页面的卡片元素。
    *   **必须** 使用 CSS `box-shadow`（模拟手绘阴影，可能需要多层模糊度低的阴影）或边框、背景纹理变化来实现层级感和悬停效果。配合 `transition` 使效果有响应。
*   **图片占位:**
    *   **核心要求:** 确保所有设计的图片区域在最终HTML中**必须可靠显示**，避免出现破损图片或空白区域。追求更佳的视觉效果和多样性。
    *   **优化策略:**
        *   **策略一 (推荐优先，兼顾美观与灵活性): 使用可靠的、基于ID的图片服务 (如 Picsum Photos)。**
            *   **替代 Unsplash:** 放弃之前固定的 Unsplash URL。改为使用 `picsum.photos` 这类服务，但**严格使用其基于特定图片ID的URL格式**，**严禁使用随机 (`/random`) 功能**。
            *   **URL 格式:** `https://picsum.photos/id/{IMAGE_ID}/{width}/{height}` (例如: `https://picsum.photos/id/1011/400/400`)。
            *   **增加多样性:** 请从以下**扩展的 Picsum ID 列表**中选择并**循环使用**，以提供更丰富的视觉内容：
                *   `10` (Forest)
                *   `20` (Cityscape)
                *   `33` (Abstract Texture)
                *   `42` (Minimalist Object)
                *   `1011` (Woman with hat)
                *   `1015` (Lake reflection)
                *   `1025` (Dog)
                *   `103` (Lighthouse)
                *   `1040` (Mountains)
                *   `1062` (Laptop on desk)
                *   `1074` (Waves)
                *   `1080` (Abstract Lights)
                *   `200` (Man looking at view)
                *   `219` (Coffee)
                *   `237` (Puppy)
                *   (你可以根据需要进一步扩充此列表，确保ID有效)
            *   **使用方法:** 在 `<img>` 标签中使用这些固定的ID链接，并设置 `alt` 属性（如 `alt="Placeholder Image from Picsum"`）。确保通过CSS控制尺寸和 `object-fit: cover;`。可考虑用CSS filter（如`sepia`, `contrast`, `saturate`）微调图片色调以更融入风格。
        *   **策略二 (最可靠的后备方案 / 适用于简单场景): 生成美观的内联 SVG 占位符。**
            *   **提升美观度:** 创建符合**手绘风格**的SVG占位符：
                *   **背景:** 使用**带有纹理图案（如用`<pattern>`定义的纸张纹理）的浅色背景**，或**模拟水彩/墨迹晕染的简单`<path>`或`<rect>`填充**。
                *   **内容:**
                    *   可以包含**简单的手绘线条图案**（如交叉线、波浪线、点阵，线条使用`<path>`并添加轻微抖动filter）。
                    *   或者绘制**简笔画风格的图标**（云、星、叶子、笑脸等），线条粗犷。
                    *   颜色使用**主文字的深色**或**柔和的强调色**，填充可以不均匀。
                    *   可以在中心放置一个**手绘风格的图标** (使用 `<use>` 引用 `icon-image` 等)。
            *   **多样化SVG:** 为不同区域生成**图案或颜色略有差异**的SVG占位符。
            *   **实现:** 直接在HTML中嵌入 `<svg>` 代码，确保其尺寸适应容器。
    *   **实施优先级:**
        *   对于需要**真实照片感**的区域（如愿景板主要图片、目标详情大图），**优先尝试策略一 (Picsum ID)**。
        *   对于**头像、列表缩略图**等小型或次要区域，或作为策略一失败时的**最终后备**，**必须使用策略二 (美观SVG)**。
    *   **最终目标:** 无论采用哪种策略，生成的 HTML 在浏览器中打开时，所有设计的图片区域都**必须有吸引人的、符合主题的可见内容**。
