## Project Overview

### Current Status
SnapRep is a Flutter + NestJS + PostgreSQL fitness application featuring:
- Complete user authentication system (Supabase Auth + JWT)
- Rich fitness features (scenario training, equipment recognition, exercise recommendations)
- Gamification system (card collection, challenge completion, theme week activities)
- User profiling analytics (preference learning, statistics tracking)

### Subscription Goals
1. **User Experience**: Guide users to pay for premium features without affecting core functionality
2. **Business Model**: Establish sustainable subscription revenue to support product development
3. **Technical Architecture**: Integrate Google Play subscriptions with cross-device sync and anti-piracy
4. **Product Strategy**: Gradually unlock premium features to cultivate user payment habits

## Technical Implementation

### 1. Database Design

#### 1.1 New Table Structure

Based on existing Prisma Schema, add the following tables:

```prisma
// =============================================================================
// SUBSCRIPTION SYSTEM - Subscription system related tables
// =============================================================================

// Subscription tier enum
enum SubscriptionTier {
  FREE       // Free version
  PREMIUM    // Premium monthly
  PREMIUM_YEARLY // Premium yearly
}

// Subscription status enum
enum SubscriptionStatus {
  ACTIVE        // Active
  PAST_DUE      // Past due
  CANCELED      // Canceled (expires at end date)
  UNPAID        // Unpaid
  EXPIRED       // Expired
}

// Payment platform enum
enum PaymentPlatform {
  GOOGLE_PLAY   // Google Play Store
  APPLE_STORE   // Apple App Store (reserved)
  STRIPE        // Stripe (reserved)
}

// Subscription record table
model Subscription {
  id     String @id @default(cuid())
  userId String @unique @map("user_id") // One user can only have one active subscription
  user   User   @relation(fields: [userId], references: [id], onDelete: Cascade)

  // Subscription configuration
  tier           SubscriptionTier   @default(FREE)
  status         SubscriptionStatus @default(ACTIVE)
  paymentPlatform PaymentPlatform?  @map("payment_platform")

  // Google Play related fields
  productId      String? @map("product_id")      // Google Play product ID
  purchaseToken  String? @map("purchase_token")  // Google Play purchase token
  orderId        String? @map("order_id")        // Google Play order ID

  // Subscription cycle
  startDate  DateTime  @map("start_date") @db.Timestamptz(6)
  endDate    DateTime? @map("end_date") @db.Timestamptz(6)     // null means unlimited
  renewsAt   DateTime? @map("renews_at") @db.Timestamptz(6)    // Next renewal time

  // Trial configuration
  trialStartDate DateTime? @map("trial_start_date") @db.Timestamptz(6)
  trialEndDate   DateTime? @map("trial_end_date") @db.Timestamptz(6)
  isTrialUsed    Boolean   @default(false) @map("is_trial_used")

  // Price information (record actual user payment, support price lock)
  currency       String @default("USD")
  originalPrice  Float  @map("original_price")  // Original price
  actualPrice    Float  @map("actual_price")    // Actual payment price

  // Cancellation related
  canceledAt       DateTime? @map("canceled_at") @db.Timestamptz(6)
  cancelReason     String?   @map("cancel_reason") @db.VarChar(500)
  willRenew        Boolean   @default(true) @map("will_renew")

  // Verification related
  lastVerifiedAt   DateTime? @map("last_verified_at") @db.Timestamptz(6)
  verificationData Json?     @map("verification_data")

  // Metadata
  createdAt DateTime @default(now()) @map("created_at") @db.Timestamptz(6)
  updatedAt DateTime @updatedAt @map("updated_at") @db.Timestamptz(6)

  // Relations
  paymentTransactions PaymentTransaction[]

  @@index([userId, status])
  @@index([status, endDate])
  @@index([paymentPlatform, status])
  @@map("subscriptions")
}

// Usage tracking table for daily exercise limits
model DailyUsage {
  id     String @id @default(cuid())
  userId String @map("user_id")
  user   User   @relation(fields: [userId], references: [id], onDelete: Cascade)

  // Usage tracking
  usageDate      DateTime @map("usage_date") @db.Date
  exerciseCount  Int      @default(0) @map("exercise_count") // Daily exercise count
  resetAt        DateTime @map("reset_at") @db.Timestamptz(6) // Reset time (midnight local time)

  // Metadata
  createdAt DateTime @default(now()) @map("created_at") @db.Timestamptz(6)
  updatedAt DateTime @updatedAt @map("updated_at") @db.Timestamptz(6)

  @@unique([userId, usageDate])
  @@index([userId, usageDate])
  @@map("daily_usage")
}
```

### 7. Paywall Trigger Strategy

#### 7.1 Trigger Timing Design

**Core Trigger Strategy**

1. **New User Trial Period Limit**
   - All new users get a 7-day free trial
   - After trial ends, subscription required to continue

2. **Daily Exercise Limit (Free Users)**
   - Free users can only complete 3 exercises per day
   - When exceeding 3 exercises, immediately show paywall
   - Reset time: Midnight local timezone

**Secondary Triggers (Medium Priority)**

1. **Value Demonstration Triggers**
   - After first workout completion, show Premium additional features
   - When viewing others' share cards, suggest Premium social features
   - When viewing basic statistics, show Premium analytics preview

2. **Trial Period Reminders**
   - Day 5 of trial: Friendly reminder of 2 days remaining
   - Day 6 of trial: Last day trial reminder
   - Trial ended: Mandatory subscription popup

#### 7.2 Trigger Logic Implementation

```dart
class PaywallTriggerService {
  static const int FREE_DAILY_EXERCISE_LIMIT = 3;

  // Check if user should see paywall
  static Future<bool> shouldShowPaywall(String userId, String triggerType) async {
    final user = await UserService().getUser(userId);

    // Skip if premium user
    if (user.subscriptionTier != SubscriptionTier.FREE) {
      return false;
    }

    switch (triggerType) {
      case 'daily_exercise_limit':
        return await _checkDailyExerciseLimit(userId);
      case 'trial_expired':
        return await _checkTrialExpired(userId);
      default:
        return false;
    }
  }

  static Future<bool> _checkDailyExerciseLimit(String userId) async {
    final today = DateTime.now().toLocal().date;
    final usage = await DailyUsageService().getTodayUsage(userId, today);
    return usage.exerciseCount >= FREE_DAILY_EXERCISE_LIMIT;
  }

  static Future<bool> _checkTrialExpired(String userId) async {
    final user = await UserService().getUser(userId);
    if (user.freeTrialUsed) return false;

    final trialEndDate = user.trialStartedAt?.add(Duration(days: 7));
    return trialEndDate != null && DateTime.now().isAfter(trialEndDate);
  }
}
```
