# SnapRep Flutter 前端开发指南

## 快速总览
- **入口**：lib/main.dart 初始化 Supabase、Google 登录，然后把多个 Provider 注入 MaterialApp。
- **路由**：lib/routes/app_routes.dart 集中定义名称、页面构建器，以及 快速开始挑战一键加入等辅助导航方法。
- **核心层**：lib/core/** 提供配置（API/Supabase）、数据模型、服务类（HTTP/Supabase/AI 等）与 Provider。
- **功能页**：lib/features/** 以业务划分（home、exercises、workout_result、challenges...），每个功能内含 screens/widgets/providers。
- **共享 UI**：lib/shared/widgets/** 存放跨页面的组件（底部导航、进度条、动作卡片等），需要定制视觉时先找这里。

## 启动与路由流程
1. **初始化**（lib/main.dart）  
   - SupabaseService.instance.initialize() 连接 Supabase 并缓存 SupabaseClient。  
   - GoogleAuthService().initialize() 准备可选的 Google 登录。  
   - MultiProvider 注册 HomeProvider、WorkoutGuideProvider 等多种状态。
2. **应用骨架**  
   - MaterialApp 设置主题、字体、AppRoutes.getRoutes() 以及 AppRouteObserver，初始路由为 AppRoutes.splash。
3. **认证包装器**  
   - AuthWrapper 会检测当前 Supabase Session，不存在时走匿名登录，结束后跳 HomePage。
4. **命名路由**  
   - AppRoutes 中的 getRoutes() 把名称映射到 WidgetBuilder，同时提供封装方法：  
     - 
avigateToWorkoutGuideStep1 / 
avigateToWorkoutResult / quickStartWorkout 等描述业务流程。  
     - AppRoutes.scenarioQuickSelect、equipmentQuickSelect 根据预设参数（例如 chair 对应 intent: STRETCH）直接跳到结果页。
5. **页面内跳转**  
   - 首页给我60秒调用 AppRoutes.navigateTo(context, AppRoutes.recommendedExercises)。  
   - 底部导航通过 BottomNavigationBarWidget 调 AppRoutes.cameraDetection、AppRoutes.myPage。  
   - ModernWorkoutResultPage 的Start Practice通过 Navigator.push 打开 ProfessionalWorkoutVideoPageV2。

## 数据与服务层
### 1. 环境与配置
- ApiConfig.baseUrl 默认指向 http://localhost:3000。如果后端部署在远端，在这里修改。
- AppConstants.nestJsApiUrl 会根据 IS_PRODUCTION 切换 http://127.0.0.1:3000 与线上域名，所有 ApiService 内的 HTTP 请求都引用它。

### 2. SupabaseService (lib/core/services/supabase_service.dart)
- 负责初始化、匿名登录、登出、读取当前用户。
- 其他服务需要 Supabase Token / 用户 ID 时，通过 SupabaseService.instance 获取。
- 例：ApiService._headers 自动附加 Authorization: Bearer <token>。

### 3. ApiService (lib/core/services/api_service.dart)
- 提供高阶业务 API：场景、器材、主题周、推荐、成果卡、用户、日历等。
- 同时访问 NestJS（使用 http 包）和 Supabase REST。  
- 示例：获取场景
  `dart
  final api = ApiService.instance;
  final scenarios = await api.getScenarios(page: 1, pageSize: 20);
  `
- 示例：生成快速推荐
  `dart
  final session = await api.generateQuickRecommendation({
    'intent': 'STRETCH',
    'equipment': ['chair'],
    'scenario': 'office',
    'duration': 60,
  });
  `
  返回 WorkoutSession 模型，可继续跳到结果卡页面。

### 4. Specialized Services
- **ExerciseService**：封装 /api/v1/recommendations/* 与 /api/v1/exercises/search。ModernWorkoutResultPage 用它来拉取三张主卡片 + 替换候选。
- **PopularExercisesService**：简单 GET，返回 PopularExerciseDto，在热门动作页演示从接口直接渲染网格。
- **ChallengesService**：挑战列表、进度、完成等接口，配合 eatures/challenges/**。
- **DefaultDataService**：当线上接口不可用时提供兜底数据。HomeProvider 会在拉取失败且本地无缓存时用它构造临时列表。

### 5. 模型 (lib/core/models/**)
- 使用 json_serializable 生成 *.g.dart，常用模型：Scenario、Equipment、Exercise、WorkoutIntent、TargetMuscle、WorkoutSession 等。
- 新增字段时别忘了运行 lutter pub run build_runner build 重新生成序列化代码。

## 状态管理与数据展示
1. **Provider 调用服务**  
   - HomeProvider 内部注入 ApiService + DefaultDataService。  
   - loadHomeData() 并发调用 loadScenarios() / loadEquipment()，设置 loading、错误信息，并在失败时回退到默认数据。
2. **Widget 订阅状态**  
   - 在页面层面使用 Consumer<HomeProvider> 或 context.watch<HomeProvider>()。
   - 首页示例（节选自 lib/features/home/screens/home_page.dart）：
     `dart
     @override
     Widget build(BuildContext context) {
       return Consumer<HomeProvider>(
         builder: (context, homeProvider, _) {
           return Column(
             children: [
               HorizontalScrollSection<Scenario>(
                 title: 'Scenarios',
                 items: homeProvider.scenarios,
                 isLoading: homeProvider.isLoadingScenarios,
                 error: homeProvider.scenariosError,
                 onItemTap: (scenario) => () => _onScenarioPressed(scenario),
                 ...
               ),
               HorizontalScrollSection<Equipment>( ... ),
             ],
           );
         },
       );
     }
     `
3. **组件负责状态展示**  
   - HorizontalScrollSection 内建 Loading Skeleton、错误态和空态，你只需提供 items + getTitle/getImageUrl/onItemTap。
   - RecommendedExercisesPage 展示服务 -> 状态 -> UI完整流程：  
     - initState 中调用 _loadPopularExercises()。  
     - 根据 isLoading / error / exercises 渲染三种状态。  
     - GridView.builder + GestureDetector 跳转到挑战页面。
4. **结果页卡片组合**  
   - ModernWorkoutResultPage 在 _loadWorkoutData 中获取推荐、备选数据，之后通过 _buildWorkoutCard、_buildPreviewStrip 渲染复杂 UI。

## UI 模块与组件
- **features/home**：英雄区、场景/器材横向列表、挑战区，提供入口到所有业务流程。
- **features/exercises**：热门推荐列表，是理解简单页面 + API的最佳示例。
- **features/workout_result**：现代风格推荐结果页，含动画、底部操作条、自定义替换 BottomSheet。
- **features/workout_guide / onboarding**：三步引导（场景 → 物品 → 意图 → 目标肌群），搭配 StepProgressIndicator 与各类选择卡。
- **features/challenges**：列表页 + 视频播放 + 进度跟踪。
- **features/profile/result_card**：生成和展示成果卡，包含大量 CustomPainter，用于丰富自定义视觉。
- **shared/widgets**：复用组件库。优先使用这些组件，可保持设计一致并减少重复逻辑。

## 绘图与自定义视觉
1. **CustomPainter 基础**  
   - lib/shared/widgets/bottom_navigation_bar.dart 里的 HomeIconPainter / CameraIconPainter / ProfileIconPainter 展示了如何在 paint(Canvas canvas, Size size) 里绘制路径、圆形、圆角矩形，再通过 CustomPaint 嵌入到 Widget。
   - 若要新增自定义图标，只需实现新的 CustomPainter，并在 _buildNavItem 中替换 painter。
2. **高级绘图案例**  
   - eatures/result_card/screens/result_card_page.dart 与 eward_card_display_page.dart 结合渐变、粒子特效（SparklesPainter、FloatingParticlesPainter）打造卡片效果。
   - lib/shared/widgets/step_progress_indicator.dart 用 Container + BoxDecoration 实现简单的线性进度，可在引导/挑战步骤中直接复用。
3. **自定义图形模板**  
   `dart
   import 'dart:math' as math; // 放在文件顶部

   class ArcProgressPainter extends CustomPainter {
     final double progress;
     ArcProgressPainter(this.progress);

     @override
     void paint(Canvas canvas, Size size) {
       final background = Paint()
         ..color = Colors.grey.shade300
         ..style = PaintingStyle.stroke
         ..strokeWidth = 8
         ..strokeCap = StrokeCap.round;

       final foreground = Paint()
         ..color = const Color(0xFFFFD700)
         ..style = PaintingStyle.stroke
         ..strokeWidth = 8
         ..strokeCap = StrokeCap.round;

       final rect = Rect.fromLTWH(0, 0, size.width, size.height);
       canvas.drawArc(rect, -math.pi, math.pi, false, background);
       canvas.drawArc(rect, -math.pi, math.pi * progress, false, foreground);
     }

     @override
     bool shouldRepaint(covariant ArcProgressPainter oldDelegate) =>
         oldDelegate.progress != progress;
   }
   `
   将其包在 CustomPaint(size: const Size(120, 60), painter: ArcProgressPainter(0.75)) 即可渲染半圆进度。

## API → UI → 路由实战
### 1. 热门推荐动作页（Home CTA → RecommendedExercisesPage）
1. 用户点击给我60秒 → HomePage._onQuickStartPressed() 调用 AppRoutes.navigateTo(..., AppRoutes.recommendedExercises)。
2. RecommendedExercisesPage 初始化时调用 PopularExercisesService.getPopularExercises()，解析为 List<PopularExerciseDto>。
3. GridView.builder 渲染卡片，GestureDetector 的 onTap 使用 Navigator.pushNamed('/challenges', arguments: {...}) 跳转到挑战页，并把当前动作信息带过去。

### 2. 场景/器材 → 引导/结果页（Scenario Chip → Workout Result）
1. HomeProvider 从 ApiService.getScenarios() 拿到 Scenario 列表，传给 HorizontalScrollSection。
2. 点击某个场景：
   `dart
   void _onScenarioPressed(Scenario scenario) {
     AppRoutes.navigateTo(context, AppRoutes.cameraDetection);
   }
   `
   或者使用 AppRoutes.scenarioQuickSelect 直接跳到推荐结果，内部会把预设参数（如 chair、intent: STRETCH）传给 
avigateToWorkoutResult。
3. ModernWorkoutResultPage 在 _loadWorkoutData() 中解析 ecommendationParams，调用 ExerciseService.getQuickRecommendation()。
4. 数据展示后，用户可点Start Practice跳转到视频页，或点New Set重新调用接口刷新。

### 3. 自己调用后端接口并展示
假设要新增最近完成的训练列表：
1. **服务层**：在 ApiService 中添加 Future<List<WorkoutSession>> getRecentSessions()，调用 Supabase workout_sessions。
2. **Provider**：创建 RecentSessionsProvider，在 load() 中调用上述方法并处理 loading/error。
3. **UI**：在某个页面 ChangeNotifierProvider(create: (_) => RecentSessionsProvider()..load())，通过 Consumer 渲染 ListView。
4. **路由**：需要新页面时，在 AppRoutes.getRoutes() 添加条目，并在 UI 中调用 AppRoutes.navigateTo(context, AppRoutes.xxx)。

## 扩展路由与新功能
1. **新增页面**
   - 在 lib/features/<feature>/screens/ 创建新文件，例如 stats_page.dart。
   - 在 AppRoutes 中添加常量、getRoutes() 映射以及可选的封装方法（便于复用导航逻辑）。
2. **新增底部导航项**
   - 修改 BottomNavigationBarWidget，增加 _buildNavItem、自定义 Painter，以及在 HomePage._onNavTap 中处理索引。
3. **接入新 API**
   - 如果需要携带用户 Token，优先在 ApiService 添加方法，这样统一复用 _headers。
   - 纯公共接口可以新建 lib/core/services/<new_service>.dart，参考 PopularExercisesService。
4. **复用组件**
   - Action 卡片、进度条、选择器等尽量复用 lib/shared/widgets 中的组件，必要时在原组件里抽象通用 props。

## 调试、排错与常见问题
- **设备与依赖**
  `ash
  flutter doctor
  cd frontend
  flutter pub get
  flutter run -d windows   # Windows 桌面调试
  flutter run -d chrome    # Web 调试
  `
- **代码生成**：模型新增字段后运行 lutter pub run build_runner build --delete-conflicting-outputs。
- **网络问题**
  - 本地联调建议把后端跑在 127.0.0.1:3000，确保手机/模拟器能访问。
  - Android 需要在 ndroid/app/src/main/AndroidManifest.xml 加 usesCleartextTraffic=true 才能访问 HTTP。
- **常见异常**
  - MissingPluginException：桌面端使用 Google 登录时出现，保持捕获并提示用户；不会阻塞其他功能。
  - HandshakeException/SocketException：确认后端可达、ApiConfig.baseUrl 正确。
  - Supabase 登录失败：AuthWrapper 已经做了容错（失败会跳过认证），但要关注日志。
- **性能与体验**
  - 尽量使用 Provider 的 
otifyListeners() 控制范围（在 Provider 内做合并更新）。  
  - 列表加载大图时使用 CachedNetworkImage，并提供 placeholder/errorWidget（参见 HorizontalScrollSection）。
  - 自定义动画使用 AnimationController，记得在 dispose() 中释放。

如需进一步扩展（例如新增可视化、调整路由、定制推荐逻辑），按照服务层 → Provider → UI → 路由的顺序逐层修改即可。先在 docs/ 记录设计，再在 lib/core/lib/features 实现，可以保持代码与文档同步更新。
