# 相机功能注释恢复指南

## 概述

由于Google Play Store要求应用程序使用敏感权限（如摄像头权限）时必须提供隐私政策，我们暂时注释了SnapRep应用的AI相机扫描功能。本文档详细记录了所有被注释的代码位置和内容，以便将来恢复这些功能。

**注释日期**: 2025年11月30日
**版本**: 1.0.0+2
**目的**: 通过Google Play Store上线审核

## 相机功能说明

### 功能描述
AI相机扫描功能允许用户通过手机摄像头自动识别锻炼环境和设备：

1. **环境识别**: 自动识别家庭、办公室、健身房等锻炼场所
2. **设备检测**: 识别可用的健身器材（如哑铃、瑜伽垫、椅子等）
3. **智能推荐**: 基于检测结果推荐合适的锻炼方案
4. **跳跃流程**: AI检测成功后直接跳转到锻炼模式选择，无需手动选择设备

### 技术实现
- 使用Flutter `camera` 插件访问设备摄像头
- 实时显示相机预览
- 模拟AI分析过程（3秒延迟）
- 动画效果包括扫描线、脉冲动画等
- 支持多种环境的模拟检测结果

### UI交互流程
1. 用户进入场景选择页面
2. 点击右上角的"AI Scan"按钮
3. 进入相机扫描模式
4. 用户点击拍照按钮开始分析
5. 显示AI分析进度动画
6. 展示检测结果（场景和设备）
7. 用户确认后自动保存并跳转

## 隐私政策要求

### Google Play要求
当应用使用以下权限时，必须提供隐私政策：
- `android.permission.CAMERA`
- `android.hardware.camera`

### 隐私政策内容应包含
1. **数据收集说明**: 明确说明摄像头仅用于本地分析
2. **数据使用目的**: 用于识别锻炼环境和设备
3. **数据存储**: 说明是否保存图像数据
4. **第三方共享**: 是否与第三方共享相机数据
5. **用户权利**: 用户如何管理相机权限

### 示例隐私政策条款
```
相机权限使用说明：
SnapRep应用请求相机权限用于AI场景检测功能，帮助用户自动识别锻炼环境和可用设备。
- 相机数据仅在设备本地处理，不会上传至服务器
- 不会保存或存储任何图像或视频文件
- 用户可以随时在系统设置中撤销相机权限
- 撤销权限后，AI扫描功能将不可用，但不影响其他功能的正常使用
```

## 重新启用相机功能的步骤

### 1. 准备隐私政策
1. 创建隐私政策网页
2. 确保政策内容符合Google Play要求
3. 获取隐私政策的公开URL

### 2. 更新Google Play Console
1. 登录Google Play Console
2. 进入应用的"Store listing"
3. 在"Privacy Policy"部分添加隐私政策URL
4. 保存更改

### 3. 恢复代码功能
```bash
# 1. 恢复相机功能代码
cp docs/backup/scenario_selection_page_with_camera.dart.backup frontend/lib/features/workout_guide/screens/scenario_selection_page.dart

# 2. 恢复Android权限
cp docs/backup/AndroidManifest_with_camera.xml.backup frontend/android/app/src/main/AndroidManifest.xml

# 3. 更新版本号
# 编辑 frontend/pubspec.yaml，增加版本号，如 1.0.0+3

# 4. 重新构建
cd frontend
flutter pub get
flutter build appbundle --release
```

### 4. 重新提交到Google Play Store
1. 上传新的AAB文件
2. 确认隐私政策已正确配置
3. 提交审核

## 注意事项

### 开发注意点
1. **测试**: 恢复功能后需要在真机上测试相机功能
2. **权限**: 确保应用正确请求和处理相机权限
3. **兼容性**: 测试不同Android版本的相机兼容性
4. **性能**: 相机功能可能影响应用启动速度

### 审核注意点
1. **隐私政策**: 必须在应用上线前配置隐私政策
2. **权限说明**: 在应用内明确说明相机权限的用途
3. **可选功能**: 确保用户可以拒绝相机权限后仍能正常使用应用
