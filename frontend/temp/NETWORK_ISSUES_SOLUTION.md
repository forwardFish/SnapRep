# Flutter Android 构建网络问题解决方案

## 问题现状
您的开发环境遇到严重的网络问题：
1. **TLS握手失败** - 无法连接Google官方源
2. **SSL证书验证失败** - 国内镜像源证书不匹配
3. **网络超时** - 依赖下载失败

## 最佳解决方案（推荐）

### 方案1：使用科学上网工具
这是最彻底和稳定的解决方案：

1. 安装科学上网工具（如Clash、V2ray等）
2. 开启系统代理或终端代理
3. 在Android Studio中配置代理：
   - File -> Settings -> Appearance & Behavior -> System Settings -> HTTP Proxy
   - 选择 "Manual proxy configuration"
   - 设置代理地址（通常是 127.0.0.1:7890）

4. 运行命令：
   ```bash
   cd frontend
   flutter run -d PQY0220C26003745
   ```

### 方案2：使用移动热点
如果无法使用科学上网：

1. 使用手机开启热点
2. 电脑连接手机热点
3. 通过移动网络访问Google服务（移动网络通常不会被墙）
4. 运行Flutter构建

### 方案3：离线构建包（临时方案）
如果以上方案都不可行：

1. 从有科学上网的朋友/同事那里获取已构建的APK
2. 或者请求他们帮忙完成首次构建，并打包 `.gradle` 缓存文件夹
3. 将缓存复制到您的 `C:\Users\您的用户名\.gradle` 目录
